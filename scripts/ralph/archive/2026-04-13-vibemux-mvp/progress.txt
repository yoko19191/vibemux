## Codebase Patterns
- Tauri commands are in `apps/desktop/src-tauri/src/commands.rs`, registered in `lib.rs` invoke_handler
- Config system in `apps/desktop/src-tauri/src/config.rs` — UserConfig with sections, load/save with atomic write, serde + TOML
- Frontend components in `apps/desktop/src/lib/` — Svelte 5 with $state/$derived/$props runes
- App.svelte is the main orchestrator: holds sessions state, event listener, keyboard handler, and renders Deck/Shelf/overlays
- MuxEvents emitted from Rust via `events.rs` event bridge, frontend listens on `mux-event` channel
- Existing context menu pattern in TerminalPane.svelte: inline overlay + positioned menu div
- DeckPane has drag-handle (3px colored bar) + pane-header (22px with session name) + terminal-container
- Config prefix key is now config-driven via `keymap.ts` parsePrefixKey/matchesPrefixKey/formatPrefixKey
- KeysConfig in config.rs has `prefix: String` field, default now "ctrl+b"
- Settings button is a floating `<button class="settings-btn">` in App.svelte bottom-right
- `config_get` returns the full UserConfig as JSON — access nested fields like `cfg.keys.prefix`
